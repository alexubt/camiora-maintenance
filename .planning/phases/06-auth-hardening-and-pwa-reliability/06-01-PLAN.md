<objective>
Add silent token refresh to the PKCE auth flow so users are never interrupted by token expiry during uploads.

Purpose: Microsoft access tokens expire after 1 hour. Field users leave the app open and get auth errors mid-upload. Adding `offline_access` scope returns a rotating refresh token (24h lifetime) that enables silent renewal.

Output: Updated auth.js with refresh capability, all Graph API callers wired to use getValidToken().
</objective>

<task type="auto" tdd="true">
  <name>Task 1: Add refresh token support to auth.js with tests</name>
  <files>app/graph/auth.js, app/graph/auth.test.js</files>
  <behavior>
    - getValidToken returns current token when expiry is >5 min away
    - getValidToken calls refreshAccessToken when expiry is <5 min away
    - getValidToken returns null when no token exists
    - refreshAccessToken POSTs grant_type=refresh_token to token endpoint and calls saveToken with response
    - refreshAccessToken returns false when no ms_refresh_token in sessionStorage
    - saveToken(tokenData) stores access_token AND refresh_token (rotating) in sessionStorage
    - signOut clears ms_refresh_token from sessionStorage
    - SCOPES includes offline_access
  </behavior>
  <action>
1. Update SCOPES to `'Files.ReadWrite User.Read offline_access'`.

2. Change `saveToken(token, expiresIn)` signature to `saveToken(tokenData)` where tokenData is the full token response object `{ access_token, expires_in, refresh_token }`. Store `tokenData.refresh_token` in `sessionStorage.setItem('ms_refresh_token', tokenData.refresh_token)`. Keep backward compat: if tokenData is a string (old callers), treat as access_token only.

3. Add `refreshAccessToken()` — async function that reads `ms_refresh_token` from sessionStorage, POSTs to `https://login.microsoftonline.com/${CONFIG.TENANT_ID}/oauth2/v2.0/token` with `{ client_id, grant_type: 'refresh_token', refresh_token, scope: SCOPES }`, calls `saveToken(data)` on success, returns boolean.

4. Add `getValidToken()` — async function that checks `state.token` exists and `state.tokenExp - Date.now() > 5 * 60 * 1000`. If within 5 min of expiry, calls `refreshAccessToken()`. Returns `state.token` or null.

5. Update `loadToken()` to also restore refresh_token existence check (no need to load it into state — keep only in sessionStorage per research pitfall #2).

6. Update `signOut()` to explicitly clear `ms_refresh_token` (sessionStorage.clear() already does this, but be explicit for clarity).

7. Update `main.js` boot: change `saveToken(tokenData.access_token, tokenData.expires_in)` to `saveToken(tokenData)` so refresh_token is captured on initial login.

8. Write tests in `app/graph/auth.test.js` using `node:test` with `mock` for sessionStorage and fetch. Test all behaviors listed above. Use the same pure-function + mock pattern as existing test files (e.g., `app/graph/csv.test.js`).

Note: Do NOT store refresh_token in `state` object — keep it only in sessionStorage to prevent stale in-memory copies (per research Pitfall #2: rotating tokens).
  </action>
  <verify>
    <automated>node --test app/graph/auth.test.js</automated>
  </verify>
  <done>SCOPES includes offline_access. saveToken stores refresh_token. getValidToken checks expiry and auto-refreshes. refreshAccessToken POSTs to token endpoint. signOut clears refresh token. All tests pass.</done>
</task>

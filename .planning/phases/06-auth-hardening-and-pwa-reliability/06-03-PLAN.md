<objective>
Add an offline upload queue so uploads started without connectivity are automatically retried when the connection returns.

Purpose: Field users are often on poor cellular networks. Without a queue, failed uploads are lost and must be manually re-done. IndexedDB persists the PDF blob and metadata for automatic retry.

Output: New uploadQueue.js module, offline guard in upload form, online event listener for queue drain.
</objective>

    - removeJob(id) deletes a specific job by id
    - dequeueAll() returns empty array when queue is empty
    - Jobs contain pdfBlob (Blob), remotePath (string), folderPath (string), csvRow (object)

   - Adds `queuedAt: new Date().toISOString()` before storing
   - Returns the generated id

   Export `dequeueAll()`:
   - Opens a readonly transaction, calls `getAll()` on the store
   - Returns array of jobs (empty array if none)

   Export `removeJob(id)`:
   - Opens a readwrite transaction, calls `delete(id)` on the store

The simpler approach: just update cache.js to version 2 with the same onupgradeneeded that creates both stores.

<task type="auto">
  <name>Task 2: Wire offline guard in upload.js and queue drain in main.js</name>
  <files>app/views/upload.js, app/main.js, sw.js</files>

1. In `app/views/upload.js`:
   - Add import: `import { enqueueUpload } from '../storage/uploadQueue.js';`
   - Add import: `import { getValidToken } from '../graph/auth.js';`
   - In `handleSubmit()`, after collecting form data (unitId, svc, date, cost, folderPath) but BEFORE the try/catch that calls ensureFolder+uploadFile:
     - Check `if (!navigator.onLine)` — if offline:
       - Build the PDF file (the `files[0]` or scanned PDF is already in `files` array)
       - For each file, call `enqueueUpload({ pdfBlob: files[i], remotePath: folderPath + '/' + fileName, folderPath, csvRow: { InvoiceId: Date.now().toString(36), UnitId: unitId, Date: date, Type: svc, Cost: cost, PdfPath: folderPath + '/' + fileName } })`
       - Show toast: "Offline — upload queued. Will retry when connected."
       - Reset form (same cleanup as success path)
       - Return early (skip the online upload path)
     - Also wrap the existing upload try/catch: if the upload fetch throws a network error (TypeError from fetch), fall through to the same enqueue logic as the offline guard.
   - Replace `state.token` reference in `appendInvoiceRecord(invoiceRow, state.token, ...)` with `await getValidToken()` (this may already be done by Plan 01 — if so, just verify it's correct).

2. In `app/main.js`:
   - Add imports: `import { dequeueAll, removeJob } from './storage/uploadQueue.js';`
   - Add import: `import { getValidToken } from './graph/auth.js';` (may already exist from Plan 01)
   - Add import: `import { ensureFolder, uploadFile } from './graph/files.js';`
   - Add import: `import { appendInvoiceRecord } from './invoice/record.js';`
   - Register online event listener AFTER DOMContentLoaded (inside the event handler):
     ```
     window.addEventListener('online', drainUploadQueue);
     ```
   - Implement `drainUploadQueue()`:
     - `const token = await getValidToken(); if (!token) return;` (don't drain without auth)
     - `const jobs = await dequeueAll();`
     - `if (!jobs.length) return;`
     - Loop serially over jobs:
       - `await ensureFolder(job.folderPath);`
       - `const file = new File([job.pdfBlob], job.remotePath.split('/').pop(), { type: 'application/pdf' });`
       - `await uploadFile(file, job.remotePath);`
       - `await appendInvoiceRecord(job.csvRow, token, state.fleet.invoicesPath);`
       - `await removeJob(job.id);`
       - Show toast: `"Queued upload completed: ${job.remotePath.split('/').pop()}"`
     - On error in loop: `console.warn(...)`, break (stop draining, retry on next online event)
   - Also call `drainUploadQueue()` at the end of the boot sequence (after fleet data loads), to catch any jobs queued in a previous session.

3. In `sw.js`:

  <action>
Create app/views/dashboard.js as a native ES module exporting a single `render(container, params)` function following the same pattern as unit-detail.js.

**Data loading:**
- Import state from '../state.js', downloadCSV/parseCSV from '../graph/csv.js', isOverdue/getDueDate/getDueMiles from '../maintenance/schedule.js'
- If !state.token, show auth prompt with link to #upload (same pattern as unit-detail.js)
- Show a loading skeleton while data loads (same pattern as unit-detail.js)
- Load maintenance.csv and condition.csv in parallel using Promise.allSettled (same as loadUnitData pattern). Do NOT load invoices -- dashboard does not need them.
- Parse all maintenance records and all condition records from the CSVs
- For each unit in state.fleet.units, find its maintenance records and condition record

**Action items section (top of dashboard):**
- Compute today as new Date().toISOString().split('T')[0]
- For each maintenance record across ALL units, compute overdue status using isOverdue(record, today, currentMiles)
- Also compute "due soon": not overdue, but getDueDate returns a date within 7 days of today, OR getDueMiles returns miles within 500 of currentMiles
- Build two arrays: overdueItems[] and dueSoonItems[], each entry containing { unitId, type, dueDate, dueMiles, record }
- Render overdue items first with red styling (background: rgba(220,53,69,0.1), border-left: 3px solid #dc3545), showing unit ID, maintenance type, and how overdue (days past due or miles past due)
- Render due-soon items next with yellow styling (background: rgba(255,193,7,0.1), border-left: 3px solid #ffc107), showing unit ID, type, and when due
- Each action item is a tappable div with onclick navigating to #unit?id={unitId}
- If no overdue or due-soon items, show a green "All caught up" message

**Unit list section (below action items):**
- Section header "Fleet" with unit count
- Use CSS class `dash-grid` for the grid layout (defined in style.css)
- Each unit rendered as a card (class `dash-card`) showing:
  - Unit ID as the card title (bold, 15px)
  - Unit Type below the title (13px, --text-2 color)
  - Status badge in top-right corner: "Overdue" (red), "Due Soon" (yellow), or "OK" (green)
  - Use the same statusBadge approach as unit-detail.js (inline span with background color and white/dark text)
- Determine unit status: check if ANY of its maintenance records are overdue => "overdue". Else if any due-soon => "due-soon". Else "ok".
- Each card is an anchor tag or clickable div navigating to #unit?id={unitId}
- Sort units: overdue first, then due-soon, then ok

**Layout:**
- Wrap everything in a div with padding:16px and padding-bottom:80px (for safe area)
- Header area: app name "Camiora" (or just skip -- the router handles this) -- actually, add a simple "Dashboard" h2
- Use escapeHtml from unit-detail.js pattern (re-implement the same 4-replace function locally, do NOT import from unit-detail.js to avoid tight coupling)

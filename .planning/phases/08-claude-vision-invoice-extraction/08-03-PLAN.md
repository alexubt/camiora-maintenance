Create `app/invoice/batch-milestone.js`:

```
import { downloadCSV, parseCSV, serializeCSV, writeCSVWithLock } from '../graph/csv.js'

const MAINTENANCE_HEADERS = ['MaintId', 'UnitId', 'Type', 'IntervalDays', 'IntervalMiles', 'LastDoneDate', 'LastDoneMiles', 'Notes']

const defaultCsvOps = { downloadCSV, parseCSV, serializeCSV, writeCSVWithLock }

export async function batchMarkDone(milestoneTypes, unitId, dateStr, miles, token, maintenancePath, csvOps = defaultCsvOps)
```

Logic:
1. If `milestoneTypes` is empty, return early (no-op)
2. Download maintenance CSV once: `const { text, hash } = await csvOps.downloadCSV(maintenancePath, token)`
3. Parse rows: `const rows = csvOps.parseCSV(text)`
4. For each type in milestoneTypes:
   - Find existing row: `rows.findIndex(r => r.UnitId === unitId && r.Type === type)`
   - If found: update `LastDoneDate = dateStr`, `LastDoneMiles = String(miles || '')`
   - If not found: push new row with generated MaintId (`Date.now().toString(36) + Math.random().toString(36).slice(2, 5)`), the unitId, type, empty intervals, dateStr, miles, empty notes
5. Serialize: `csvOps.serializeCSV(MAINTENANCE_HEADERS, rows)`
6. Write once: `await csvOps.writeCSVWithLock(maintenancePath, hash, newText, token)`

Use DI pattern (csvOps parameter) for testability — matches project convention in record.js and unit-detail.js.

Create `app/invoice/batch-milestone.test.js` using node:test:
- Mock csvOps with tracking functions (count calls, capture arguments)
- Provide test CSV data with known UnitId/Type rows
- Assert single download call, single write call, correct row mutations

    - Test 4: extractInvoice throws on non-ok response with error message from body

    - Test 6: resizeImageBlob returns a Blob (skip in Node — test structure only, mark as browser-only)
  </behavior>
  <action>
Create `app/invoice/extract.js`:

1. `WORKER_URL` constant: `'https://camiora-api-proxy.camiora.workers.dev'`

2. `export async function extractInvoice(blob, mimeType, fleet)`:
   - If mimeType is `image/jpeg` or `image/png`, call `resizeImageBlob(blob)` first to cap at 1500px
   - Convert blob to raw base64 via `blobToBase64()`
   - POST to `${WORKER_URL}/extract-invoice` with `{ image: base64, mimeType, fleet }`
   - On non-ok response, parse error body and throw Error with `err.error || status`
   - Return parsed JSON response

3. `async function blobToBase64(blob)`:
   - Use FileReader.readAsDataURL, then split on comma to strip prefix
   - Return raw base64 string

4. `async function resizeImageBlob(blob, maxPx = 1500)`:
   - Use createImageBitmap + canvas to resize if larger than maxPx on either dimension
   - Return resized JPEG blob at 0.85 quality
   - Close ImageBitmap after use; zero-out canvas dimensions to release GPU memory

Create `app/invoice/extract.test.js` using node:test:
- Mock globalThis.fetch to intercept POST requests
- Verify request body structure
- For blobToBase64: since FileReader is browser-only, test the prefix-stripping logic with a helper or mark as browser-only
- For resizeImageBlob: structure-only test (createImageBitmap not in Node), document as browser-verified
  </action>

  </verify>
  <done>extract.js exports extractInvoice; tests verify request format and response handling; base64 encoding strips data-URL prefix</done>

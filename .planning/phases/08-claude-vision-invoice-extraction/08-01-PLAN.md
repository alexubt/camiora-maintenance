  <name>Task 1: Rename Worker and add /extract-invoice POST route</name>

  <action>
1. Copy `worker/samsara-proxy.js` to `worker/api-proxy.js` (do NOT delete the original yet — let the user decommission it after deploying the new one).

2. In `worker/api-proxy.js`:
   - Update the JSDoc header comment to reflect "API proxy (Samsara + Invoice Extraction)"
   - Update `corsHeaders()` to allow `'GET, POST, OPTIONS'` in Access-Control-Allow-Methods
   - After the origin gate block and before the existing `if (request.method !== 'GET')` guard, add route dispatch:
     ```
     const url = new URL(request.url);
     if (request.method === 'POST' && url.pathname === '/extract-invoice') {
       return handleExtractInvoice(request, env, origin, allowed);
     }
     ```
   - Move `const url = new URL(request.url)` above the POST check (it is currently used later for the /vehicles/stats check — reuse the same variable)
   - Keep all existing Samsara GET logic unchanged

3. Implement `handleExtractInvoice(request, env, origin, allowed)`:
   - Parse request body as JSON; return 400 `{ error: 'invalid_json' }` on failure
   - Validate `image` and `mimeType` exist; return 400 `{ error: 'missing_fields' }` if not
   - Build content block: if `mimeType === 'application/pdf'` use `type: 'document'`; otherwise use `type: 'image'` with the provided mimeType
   - Build the extraction prompt with fleet roster as comma-separated list (see CONTEXT.md for exact prompt text including milestone type mapping hints)
   - Call `https://api.anthropic.com/v1/messages` with:
     - `x-api-key: env.ANTHROPIC_API_KEY`
     - `anthropic-version: '2023-06-01'`
     - `model: 'claude-haiku-4-5-20251001'`
     - `max_tokens: 1024`
     - messages array with image/document block + text prompt block
   - If Anthropic response is not ok, return 502 `{ error: 'upstream_error', detail: <response text> }`
   - Extract text from `data.content[0].text`
   - Parse JSON: try `JSON.parse(rawText)` first; if that fails, try regex `rawText.match(/```(?:json)?\s*([\s\S]*?)```/)` to extract from markdown code block; if that also fails, return 502 `{ error: 'extraction_failed', message: 'not valid JSON' }`
   - Return 200 with the parsed extraction object
   - All responses include `corsHeaders(origin, allowed)`

4. Update `worker/wrangler.toml`:
   - `name = "camiora-api-proxy"`
   - `main = "api-proxy.js"`
   - Add comment for ANTHROPIC_API_KEY secret alongside existing SAMSARA_API_KEY comment

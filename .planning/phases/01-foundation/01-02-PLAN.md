    **4. Create `app/router.js`** — New module:

    ```javascript
    import { render as renderUpload } from './views/upload.js';

    const ROUTES = {
      '#upload': renderUpload,
    };

    export function initRouter(container) {
      const go = () => {
        const hash = window.location.hash || '#upload';
        const key  = hash.split('?')[0];
        const fn   = ROUTES[key] || renderUpload;
        container.innerHTML = '';
        fn(container);
      };
      window.addEventListener('hashchange', go);
      go(); // initial render
    }
    ```

    **5. Create `app/main.js`** — New boot module:

    - Import `{ loadToken, exchangeCodeForToken, saveToken }` from `'./graph/auth.js'`
    - Import `{ downloadCSV, parseCSV }` from `'./graph/csv.js'`
    - Import `{ getCachedFleet, setCachedFleet }` from `'./storage/cache.js'`
    - Import `{ state }` from `'./state.js'`
    - Import `{ initRouter }` from `'./router.js'`

    Boot sequence (DOMContentLoaded):
    1. Register service worker: `navigator.serviceWorker.register('sw.js').catch(() => {})`
    2. Check URL for OAuth code: `new URLSearchParams(window.location.search).get('code')`
    3. If code exists: `history.replaceState(null, '', window.location.pathname)`, exchange for token, save token
    4. If no code: call `loadToken()`
    5. Get `#app` container: `document.getElementById('app')`
    6. Call `initRouter(container)` — this renders the upload view which checks `state.token` to show auth or app
    7. If `state.token` exists, call `loadFleetData()` in background (do not await — let UI render first)

    `loadFleetData()` async function (defined in main.js):
    1. Try `getCachedFleet()` — if result, set `state.fleet.units` and `state.fleet.unitsHash`
    2. Try `downloadCSV(state.fleet.unitsPath, state.token)` — if text not null, parse with `parseCSV`, update state, call `setCachedFleet({ units, hash })`
    3. On 404 (text is null): set `state.fleet.units = []`, log to console but do not crash
    4. On network error: if cached data was loaded, silently continue; if no cache, log warning

    IMPORTANT: `loadFleetData` does NOT re-render the view. In Phase 1, the upload form does not yet display a CSV-based unit dropdown (that is wired in Phase 3). The data is loaded into state for readiness.

  <what-built>
    The 910-line app.js monolith has been split into ES modules:
    - app/graph/auth.js (PKCE authentication)
    - app/graph/files.js (OneDrive folder/upload)
    - app/graph/csv.js (CSV download/parse/lock)
    - app/views/upload.js (upload form UI)
    - app/router.js (hash-based SPA router)
    - app/main.js (boot sequence)
    - app/state.js (shared state)
    - app/storage/cache.js (IndexedDB cache)

    index.html now loads app/main.js as type="module".
    sw.js caches all new module files as v4.
  </what-built>
  <how-to-verify>
    Start a local server and test in browser:

    **Check 1 - No load errors:**

    - The button should be clickable (no dead click)

    **Check 3 - Sign in flow:**
    - Click "Sign in with Microsoft"
    - You should be redirected to Microsoft login
    - After login, you should return to the app and see the upload form

    **Check 4 - Upload form functionality:**

    - Unit number input accepts text
    - Service type dropdown works
    - Date picker works
    - "Scan document" zone is tappable/clickable
    - "Pick existing PDF" file input works
    - Preview box appears when form is filled
    - "Sign out" button clears session and shows login screen

    **Check 5 - Hash router:**
    - URL should show `#upload` or be empty (defaulting to upload)
    - Manually typing `#upload` in URL bar should keep the upload form visible

    **Check 6 - Service worker:**
    - In DevTools > Application > Service Workers: sw.js should be active
    - Cache Storage should show `camiora-v4` with all module files listed
